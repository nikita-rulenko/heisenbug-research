## Наблюдения и мысли

### 6. GLM 4.7 как evaluator — находка и проблема одновременно
Потратили прилично времени на дебаг: GLM возвращает JSON то в `content`, то в `reasoning`, то обрезает на полуслове. Починили через `_extract_scores_json()` + увеличение токенов до 8000. Для доклада — отличный кейс: "даже инфраструктура бенчмарка требует отладки".

### 7. Cerebras — бесплатный, но непредсказуемый
gpt-oss-120b с temperature=0.2 всё равно даёт дикие выбросы: S1 (генерация unit-теста) скачет от 0 до 15 между прогонами. Это не баг подхода — это свойство LLM. Для доклада: **любой бенчмарк AI-инструментов без повторных прогонов — мусор**.

### 8. Контекст 20K chars — это мало для реального проекта
175 тестов — маленький проект. Реальный Go-сервис — 1000+ тестов, 200K+ chars. При масштабе:
- MD-файлы точно не влезут в context window
- Mem0/Helixir с семантическим поиском начнут выигрывать сильнее
- Graphiti с temporal graph (если бы работал) дал бы разницу на S9

Тезис: "мы показали что на малом масштабе разницы почти нет — но это и есть ловушка".

### 9. "А почему не OpenAI?"
Вопрос будет 100%. Ответ: Cerebras бесплатный, open-weight модели, воспроизводимость. При OpenAI Graphiti и Mem0 работали бы лучше — но мы тестировали доступный стек.

### 10. v1 vs v2 — рассинхрон рейтингов
v1: Helixir 78.4% (1-е). v2: Mem0 65.1% (1-е), Helixir 63.0% (4-е). Причины: self-evaluation в v1 завышает, один прогон = лотерея, v2 = только Part A. Это демонстрация зрелости методологии — хороший слайд.

### 11. Killer-слайд — таблица sigma
```
MD-файлы:    sigma = 2.08   <- предсказуемый baseline
Mem0:        sigma = 10.97  <- семантика помогает, но нестабильно
GH Issues:   sigma = 12.66  <- фрагментация = шум
Helixir:     sigma = 16.70  <- graph retrieval добавляет стохастичность
Graphiti:    sigma = 16.17  <- raw fallback = мусор
```
Вывод: "Чем сложнее retrieval — тем выше дисперсия. Выбирайте не лучший median, а приемлемый sigma".
