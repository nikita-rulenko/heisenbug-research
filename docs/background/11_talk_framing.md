# Framing доклада: аргументация и позиционирование

## Центральный тезис

**"Контекст AI-агентов для тестирования: от хранения фактов к связыванию причин"**

Хранение контекста (MD файлы, Issues, Mem0) отвечает на вопрос "что мы знаем?".
Связывание контекста (каузальные цепочки) отвечает на вопрос "почему мы это знаем и к чему это ведёт?".

Готового production-решения для связывания нет. Доклад показывает эволюцию проблемы и направление следующего шага.

---

## Аргументация за каждый подход

### MD файлы: "конституция" для агента

**За что**: незаменимый baseline. 20-30 строк policy, которые должны быть в каждом промпте: test framework, commands, conventions, запреты. Zero cost, zero latency, version-controlled.

**Ограничение**: статичность. Не учится, не масштабируется, не хранит history. При росте до 500+ тестов — недостаточно.

**Роль в стеке**: Policy layer. Всегда первый шаг. Без него всё остальное бессмысленно — агент должен знать базовые правила прежде чем помнить факты.

**Аудитории сказать**: "Если у вас ещё нет AGENTS.md с тестовыми конвенциями — начните с этого. 10 минут, бесплатно, сразу работает."

---

### GitHub Issues MCP: координация, а не память

**За что**: bidirectional audit trail. CI failure → issue → agent reads → fixes → PR → closes. QA и AI в одном пространстве. Лучший human interface.

**Ограничение**: не обобщает паттерны. Каждый failure разбирается с нуля. При росте тикетов — та же проблема context overload. Много комментариев ≈ длинный MD файл.

**Роль в стеке**: Coordination layer. Не где хранить знания, а кто что делает. Task assignment, не knowledge base.

**Аудитории сказать**: "Issues — это workflow tool, не memory. Используйте для трекинга задач агенту, но не ожидайте что агент 'помнит' из issues."

---

### Mem0: production-ready семантическая память

**За что**: лучшее production-решение для хранения тестовых паттернов. Semantic search, auto-extraction, 90% сокращение токенов (LOCOMO benchmark). Cloud Free tier = 10K memories, достаточно для старта. Интеграция с CrewAI, LangChain, AWS Agent SDK.

**Ограничение**: 
- Self-hosted = 2-3 компонента (Vector DB + LLM, опционально Graph DB для Mem0G), нужен DevOps
- Факты атомарны и плоски — нет онтологической типизации
- При противоречии — LLM удаляет старый факт, нет audit trail
- Граф = entity-relationship (кто/что), не каузальный (почему)
- Custom extraction prompts не поддерживаются (GitHub issue #3299)

**Роль в стеке**: Knowledge layer. "Что мы знаем о тестах" — best practices, шаблоны, паттерны покрытия. Идеально для семантического retrieval.

**Аудитории сказать**: "Mem0 — очевидный выбор для production. Начните с Cloud Free. Если нужен self-hosted — будьте готовы к DevOps overhead. Для 80% задач тестирования — достаточно."

---

### Helixir: proof of concept, не продукт

**Позиционирование**: первый каузальный фреймворк AI-памяти. Не решение для production. Proof of concept, что связывание контекста (причины, эволюция, противоречия) технически возможно.

**Почему создан**: Mem0 решает задачу хранения. Но при работе с тестами постоянно возникает вопрос "почему тест упал в прошлый раз?" — а Mem0 хранит только последнюю версию факта, предыдущие причины потеряны. Нужен был инструмент, который хранит не факты, а связи между фактами.

**Что показывает**:
- Каузальные цепочки (IMPLIES, BECAUSE, CONTRADICTS, SUPERSEDE) технически работают
- Онтологическая типизация фактов (skill, preference, goal, fact, opinion, experience, achievement) даёт агенту структуру для retrieval
- FastThink (isolated scratchpad) предотвращает загрязнение памяти промежуточными мыслями
- Append-only graph сохраняет полную историю эволюции знаний

**Честные ограничения**:
- v0.1.1, 74 stars, 1 maintainer — это не production
- LLM required (Cerebras / OpenAI / Ollama) — токены на extraction
- Fallback на local LLM аффектит скорость на порядок
- Graph depth = 1 — осознанный компромисс, ограничивает глубину цепочек
- No deletion = заполнение диска при долгом использовании
- Нет benchmarks на стандартных тестах
- Нет production case studies

**Роль в стеке**: Reasoning layer (экспериментальный). "Почему мы так решили, как эволюционировали практики, что чему противоречит."

**Аудитории сказать**: "Я не рекомендую использовать Helixir в production. 1 maintainer — это несерьёзно для production зависимости. Но это направление, в которое может прийти AI-память. Сегодня мы храним факты. Завтра будем хранить связи между фактами. Helixir — первый пробный шар, proof of concept что так можно."

## Ключевые messages для аудитории

### 1. Проблема масштаба реальна
Тестов всегда больше чем production-кода (Miranda et al., 2025: 1:1 - 1:3). Context window не масштабируется (OAJAIML, 2026: MECW << заявленный). Это не решится увеличением context window.

### 2. Четыре слоя, а не четыре альтернативы
MD (policy) → Issues (coordination) → Mem0 (knowledge) → каузальная память (reasoning). Не "выберите один", а "добавляйте следующий по мере роста боли".

### 3. Начните в понедельник
- AGENTS.md с тестовыми конвенциями = 10 минут
- GitHub MCP в Cursor = 5 минут
- Mem0 Cloud Free = 5 минут
- Три из четырёх слоёв доступны бесплатно и сегодня

### 4. Хранение ≠ связывание
State-of-the-art (Mem0) хранит факты. Следующий шаг — связывать факты причинами. Готового production-решения нет. Helixir показывает что технически это возможно.

### 5. Это открытый вопрос
Доклад не закрывает тему. Доклад открывает дискуссию: как должна выглядеть AI-память для тестирования через 2 года?

## Мета-позиционирование автора

Доклад продаёт не Helixir. Helixir — это эксперимент одного человека без финансирования, продавать его как продукт очевидно глупо.

Доклад продаёт **автора как исследователя**, который мыслит достаточно широко, чтобы задаваться вопросами за рамками текущих решений рынка. Ценность — не в конкретном инструменте, а в постановке болезненного вопроса: "хранение фактов ≠ связывание фактов, и никто пока не решает вторую задачу в production".

Helixir — proof of concept, первый пробный шар. Демонстрация что каузальная AI-память технически возможна. Не рекомендация к использованию.

Практическая часть (demo) будет проведена отдельно.
